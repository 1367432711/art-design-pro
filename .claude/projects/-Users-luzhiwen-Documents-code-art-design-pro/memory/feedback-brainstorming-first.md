---
name: 新功能必须先 brainstorming 再实现
description: 所有新功能开发都必须先走 brainstorming 流程，设计批准后才能实现
type: feedback
---

**规则**: 所有新功能都必须先 brainstorming → 设计文档 → 用户批准 → 实现计划 → 编码

**为什么**: 用户希望确保每个新功能都经过充分思考和设计，避免盲目实现导致返工

**如何应用**:

- 用户提出新功能需求时，不要直接写代码
- 必须先探索需求、问澄清问题、提供方案
- 写设计文档保存到 `docs/superpowers/specs/`
- 用户批准设计后，再调用 writing-plans 技能创建实现计划
- 计划批准后再开始编码

**触发词**:

- "我想添加 XX 功能"
- "帮我设计 XX"
- "要做一个 XX 模块"
- "新建 XX 页面"
- 任何新功能请求

**示例**:

- ✅ 用户："我想做一个订单管理模块" → 启动 brainstorming 流程
- ❌ 用户："我想做一个订单管理模块" → 直接开始写代码
